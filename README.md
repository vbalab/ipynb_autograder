# AutoGrader Assistant

## Описание проекта 

Проект посвящён автоматизации проверки домашних заданий в формате Jupyter Notebook. Сейчас преподавателям и ассистентам приходится вручную сравнивать решения студентов с эталонными ноутбуками, что занимает много времени и плохо масштабируется при большом количестве студентов. Идея проекта — упростить и ускорить этот процесс, сделав проверку более единообразной и менее подверженной ошибкам.

Система принимает на вход два ноутбука: эталон и решение студента. Они автоматически разбираются, очищаются от нерелевантных блоков и анализируются: сравнивается структура, код и результаты выполнения, включая числовые метрики и текстовые выводы. Эталонное решение разбивается на набор подзадач с описанием и максимальным баллом, что позволяет проверять работу по частям, даже если реализация студента отличается по стилю или именам переменных.

На выходе система формирует понятный отчёт с оценкой по каждой задаче и комментариями о найденных недочётах, а также сводной таблицей результатов. Отчёт генерируется в виде PDF, а для удобного взаимодействия с инструментом предусмотрен бот, через который можно загружать ноутбуки и получать результаты проверки.

## Стурктура проекта

```bash
.
├── Dockerfile                  # Docker-образ приложения
├── docker-compose.yaml         # Локальный запуск и сервисы
├── pyproject.toml              # Метаданные проекта и инструменты
├── requirements.txt            # Runtime-зависимости
├── requirements-dev.txt        # Зависимости для разработки
├── README.md                   # Документация проекта
├── DEVELOPER.md                # Документация для разработчиков
├── LICENSE                     # Лицензия
├── prompts/                    # Промпты для LLM
│   ├── grader.md               # Промпт оценивания
│   └── structure.md            # Промпт структуры ответа
└── src/
    └── grader/                 # Основной Python-пакет
        ├── __main__.py         # Точка входа (python -m grader)
        ├── bot/                # Telegram/чат-бот и обработка сообщений
        ├── core/               # Конфигурация, логирование, константы
        ├── db/                 # База данных, модели, репозитории
        ├── llm/                # Работа с LLM и логикой грейдинга
        ├── services/           # Служебное для бота
        └── __init__.py
```

## Запуск

Создать и заполнить `.env` по образцу `.env.example`.

```bash
sudo systemctl start docker

docker compose build #--no-cache
docker compose up --detach --remove-orphans

docker compose stop
docker compose down
```
